配置工具主要用来管理复杂的配置对象，屏蔽文件格式，加密方式，底层存储的不同，通过一致的接口管理配置，简单的文件直接通过 IDE 编辑即可

cfg 的操作是基于一个基础配置文件的 `base.json`

```json
{
  "decoder": {
    "type": "Json"
  },
  "provider": {
    "type": "Local",
    "localProvider": {
      "filename": "test.json"
    }
  },
  "cipher": {
    "type": "Group",
    "cipherGroup": [{
      "type": "AES",
      "aesCipher": {
        "base64Key": "IrjXy4vx7iwgCLaUeu5TVUA9TkgMwSw3QWcgE/IW5W0="
      }
    }, {
      "type": "Base64",
    }]
  }
}
```

可以通过 `get` 命令查看配置内容

```sh
cfg --camel-name --in-base-file base.json -a get
cfg --camel-name --in-base-file base.json -a get --key mysql
cfg --camel-name --in-base-file base.json -a get --key mysql.password
```

配置变更前可以使用 `diff` 命令查看变更后的内容，默认情况下，val 会被尝试当做 json 解析，如果解析失败，val 会被当做一个普通字符串，如果不希望当成 json 也可以显式地通过 --raw-val 指定

```sh
cfg --camel-name --in-base-file base.json -a diff --key mysql.username --val hatlonely
cfg --camel-name --in-base-file base.json -a diff --key mysql.password --raw-val 12345678
cfg --camel-name --in-base-file base.json -a diff --key mysql --val '{
    "connMaxLifeTime": "60s",
    "database": "testdb2",
    "host": "127.0.0.1",
    "maxIdleConns": 10,
    "maxOpenConns": 20,
    "password": "",
    "port": 3306,
    "username": "hatlonely"
}'
cfg --camel-name --in-base-file base.json -a diff --key newKey --raw-val '{
    "connMaxLifeTime": "60s",
    "database": "testdb2",
    "host": "127.0.0.1",
    "maxIdleConns": 10,
    "maxOpenConns": 20,
    "password": "",
    "port": 3306,
    "username": "hatlonely"
}'
```

确认 `diff` 后，使用 `set` 命令更新配置

```sh
cfg --camel-name --in-base-file base.json -a set --key mysql.username --val hatlonely
cfg --camel-name --in-base-file base.json -a set --key mysql.password --raw-val 12345678
cfg --camel-name --in-base-file base.json -a set --key mysql --val '{
    "connMaxLifeTime": "60s",
    "database": "testdb2",
    "host": "127.0.0.1",
    "maxIdleConns": 10,
    "maxOpenConns": 20,
    "password": "",
    "port": 3306,
    "username": "hatlonely"
}'
cfg --camel-name --in-base-file base.json -a set --key newKey --raw-val '{
    "connMaxLifeTime": "60s",
    "database": "testdb2",
    "host": "127.0.0.1",
    "maxIdleConns": 10,
    "maxOpenConns": 20,
    "password": "",
    "port": 3306,
    "username": "hatlonely"
}'
```

写操作会自动备份配置，使用 rollback 可以直接回滚配置

```sh
cfg --camel-name --in-base-file base.json -a rollback --backup-file cfg.backup.json.20201113.224234
```

`put` 命令可用来加密字段

- 通过 `--set-cipher-keys` 设置需要加密的 key
- 通过 `--add-cipher-keys` 添加需要加密的 key
- 通过 `--no-cipher` 解密所有的 key

```sh
cfg --camel-name --in-base-file base.json -a put --set-cipher-keys mysql.password,redis.password
cfg --camel-name --in-base-file base.json -a put --add-cipher-keys email.password
cfg --camel-name --in-base-file base.json -a put --no-cipher
```

`put` 也可以用来迁移配置，同样需要准备一份目标的基础配置文件 `base_ots.json`

```json
{
  "decoder": {
    "type": "Json"
  },
  "provider": {
    "type": "OTS",
    "otsProvider": {
      "endpoint": "https://hatlonelu.cn-shanghai.ots.aliyuncs.com",
      "instance": "hatlonely",
      "accessKeyID": "xx",
      "accessKeySecret": "xx",
      "table": "TestConfig",
      "key": "test",
      "interval": "10m"
    }
  },
  "cipher": {
    "type": "Group",
    "cipherGroup": [{
      "type": "AES",
      "aesCipher": {
        "base64Key": "IrjXy4vx7iwgCLaUeu5TVUA9TkgMwSw3QWcgE/IW5W0="
      }
    }, {
      "type": "Base64"
    }]
  }
}
```

```sh
cfg --camel-name --in-base-file base.json -a diff --out-base-file base_ots.json
cfg --camel-name --in-base-file base.json -a put --out-base-file base_ots.json
```
